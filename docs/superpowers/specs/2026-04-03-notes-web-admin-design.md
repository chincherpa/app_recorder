# Notes Web Admin вЂ” Design Spec
**Date:** 2026-04-03  
**Status:** Approved

## Overview

A simple, unauthenticated web admin panel for reading and managing voice notes stored in the existing Supabase database. Deployed on Vercel. Lives in a separate GitHub repository (`next-notes-admin`).

## Stack

- **Framework:** Next.js (App Router)
- **Database:** Supabase (existing project at `hlluhoxrvvqmubwlorxm.supabase.co`)
- **Supabase client:** `@supabase/supabase-js` (client-side, anon key)
- **Hosting:** Vercel
- **Styling:** Tailwind CSS (included with Next.js scaffold)

## Data Model

Existing Supabase table `notes`:

| Column       | Type      | Notes                        |
|--------------|-----------|------------------------------|
| `id`         | bigint    | Auto-generated primary key   |
| `content`    | text      | The note text                |
| `created_at` | timestamp | Auto-generated by Supabase   |

## Features

### Read
- On page load, fetch all notes ordered by `created_at` descending (newest first).
- Show a loading indicator while fetching.
- Show an error message if the fetch fails.

### Create
- Text input + "HinzufГјgen" button at the top of the page.
- On submit: insert into `notes` via Supabase, then refresh the list.
- Clears the input field after successful insert.

### Update (inline edit)
- Each `NoteRow` has an "Bearbeiten" button.
- Clicking it turns the text into an editable `<textarea>`.
- Save with "Speichern" button or `Enter` key.
- Cancel with "Abbrechen" button or `Escape` key.
- On save: `UPDATE notes SET content = ... WHERE id = ...`, then refresh.

### Delete
- Each `NoteRow` has a "LГ¶schen" button.
- No confirmation dialog (personal tool, single user).
- On click: `DELETE FROM notes WHERE id = ...`, then refresh.

## Error Handling

- Loading state shown while any async operation is in progress.
- Error messages rendered as visible text below the relevant UI element.
- No retry logic вЂ” user can manually retry by re-triggering the action.

## Authentication

None. The URL serves as the only access control. The Supabase anon key is used, matching the existing Android app setup.

## Deployment

1. Push repo to GitHub.
2. Import project in Vercel dashboard.
3. Set environment variables: `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. Deploy.

## Out of Scope

- Password protection / auth
- Realtime updates (no WebSocket subscription)
- Sorting/filtering/search
- Pagination (assumed small dataset)
